Cardinality of names

Rule

A singular name MUST be used for a single object, while a plural name MUST be used for an array or collection.

Explanation

The cardinality of the name must match the object. A singular name represents the object itself, while a plural name represents a collection of objects of the same type.

Examples

Single objectCollection
personpeople
catcats
TVTVs

Language-specific notes

SQL

Due to object-relational mapping, table names in database are considered class names and column names are considered member variable names. Therefore, singular form in PascalCase is used for database tables.
