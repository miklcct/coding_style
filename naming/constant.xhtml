Constant

Rule

Constant names MUST be in SCREAMING_SNAKE_CASE.

Explanation

All constants must be in SCREAMING_SNAKE_CASE. Constants refer to compile-time constants, not immutable variables.

This rule applies to enum members as well because they are effectively constant.

Language-specific notes

C / C++

This rule does not apply in C / C++ because SCREAMING_SNAKE_CASE is reserved for macros. Use the rule for variables to name constants in C / C++.

PHP

Although TRUE, FALSE and NULL are predefined constants, true, false and null are also reserved words which are case-insensitive. They are resolved at compiled time rather than runtime like magic constants. Therefore, they MUST be specified using lower case as true, false and null.

Magic constants, which are resolved at compiled time rather than runtime, are case insensitive. They are actually not constants which cannot be accessed via defined() and do not appear in get_defined_constants(). These MUST be specified using the case listed in the manual.

Haskell

This rule does not apply in Haskell because it is purely functional and does not distinguish between constants and variables.
