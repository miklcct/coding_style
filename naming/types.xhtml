Types

Rule

User-defined type names MUST be in PascalCase.

Explanation

All user-defined types must be in PascalCase. These includes classes, interfaces, aliases, type variables, etc. as long as it represents a data type.

Language-specific notes

C / C++

This rule applies to class, struct, union, enum and typedef names, and also type aliases formed by using key word, even when the typedef or the type alias refers to a primitive type.

This rule also applies to type parameters in templates as well.

Java

This rule applies to class and interface names.

Javascript

This rule applies to function names intended as a constructor, and also ES6 class names.

PHP

This rule applies to class, interface and trait names. However, the key words self, static and parent should be entered in lower case.

Haskell

This rule does not apply to type variables to Haskell because it conflicts with the language rule.
