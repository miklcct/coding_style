Non-members

Rule

Non-member variable or function names MUST be in snake_case.

Explanation

All variables or functions outside an object must be in snake_case. These includes global variables, local variables, name space variables, function parameters, etc.

Exception: Constructor parameters assigned to member fields directly may use camelCase.

Language-specific notes

C / C++

This rule applies to non-member constants as well in addition to variables and functions, because SCREAMING_SNAKE_CASE is reserved for macros.

This rule also applies to non-type template parameters as well.

Javascript

If the member function is intended to be used as a constructor, the rule for types instead of this applies.
