Length of names

Rule

The length of names SHOULD be proportional to its scope.

Explanation

If a name is used in a large area, it should be as descriptive as possible. For example, get_database_connection is a suitable name for a global function of an application.

In contrast, short names such as count are appropriate in local names used only inside a small function, or in the extreme case of the counter of a one-line loop, even a single-letter name such as i is acceptable.
