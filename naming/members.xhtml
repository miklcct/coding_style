Members

Rule

Member variable (field) or function (method) names MUST be in camelCase.

Explanation

All member variables and functions inside an object must be in camelCase.

Language-specific notes

C / C++

This rule applies to member constants as well in addition to variables and functions, because SCREAMING_SNAKE_CASE is reserved for macros.

Javascript

If the member function is intended to be used as a constructor, the rule for types instead of this applies.
