Natural language

Rule

All names in a single project MUST be in a single language which is used by the target audience. For global projects, English must be used.

Explanation

A project must be written in a single natural language. Mixing different natural languages in names is not allowed.

If the project is not limited to specific regions, English must be used.

Examples

A project developed solely for use in Hong Kong must be in either traditional Chinese or English. Although the interface should support both languages, the names in the code must be in either one but not both.

A project developed for use only in the CIS must be in Russian, although the interface should also support other languages used in CIS, such as Kazakh, Azerbaijani, Kyrkyz, etc.

A project developed for use only in Singapore must be in English, although the interface should support all four languages in Singapore.

A project without regional limit must be in English, with the interface supporting as many languages as possible.
